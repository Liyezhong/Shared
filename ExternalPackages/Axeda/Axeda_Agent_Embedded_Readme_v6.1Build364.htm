Introduction

Build 364 of the 6.1 release of Axeda Agent Embedded resolves the following issue:

IDs

Description

3331

Agent Embedded crashes when runtime version of OpenSSL differs from one used when users built their AE applications.

 

CAUSE: When checking the strength of the cipher after establishing an SSL connection, Agent Embedded bypasses the OpenSSL API and accesses a member of the data structure for an SSL session directly. This behavior causes Agent Embedded programs to crash when the runtime version of OpenSSL differs from the version used when users build their Agent Embedded applications.

 

RESOLUTION: Agent Embedded must use OpenSSL API and not access OpenSSL data structure members directly. This build resolves this issue.

 In addition to resolving these issues, this build includes the following fixes from previous builds of version 6.1:

IDs

Description

Build 360

2989

Agent Embedded crashes with segmentation fault when processing file upload

 

When OnFileUploadBegin or OnFileUploadBeginEx callbacks are installed by the application, but return AeFalse to indicate that they will not handle the file upload, Agent Embedded crashes during file upload.

 

CAUSE: Agent Embedded did not process the return value from OnFileUploadBegin and OnFileUploadBeginEx callbacks correctly.

 

RESOLUTION: Agent Embedded has been changed to correctly process file upload internally, when OnFileUploadBegin or OnFileUploadBeginEx callbacks return AeFalse.  

2991

Agent Embedded closes file descriptor 0 when server hostname cannot be resolved

 

When Agent Embedded fails to connect to the Enterprise server due to a hostname resolution error, file descriptor 0 (which is usually associated with application’s standard input) may get closed as a result.

 

CAUSE: Agent Embedded incorrectly called AeNetDisconnect() with the argument pointing to file descriptor 0, whenever a hostname resolution error occurred.

 

RESOLUTION: Agent Embedded has been changed to not call AeNetDisconnect() when the Enterprise server connection fails due to a hostname resolution error.

Build 355

1713

Modify Agent Embedded to pass data items differently

 

Expression rules and alarms set to trigger on data item values were not working as required.

 

CAUSE: Agent Embedded could pass data items individually, but not as a set.

 

RESOLUTION: A new function has been added to Agent Embedded that allows it to post multiple data items as a single unit. This function is AeDRMPostDataItemSet. It is described in the Functions Reference chapter of the updated Axeda® Agent Embedded Reference Guide (PDF) that accompanies this build.  

New features from previous builds of version 6.1 include:

·        Support for the Restart instruction and the backup flag for the Download instruction in Software Management packages in the Axeda® ServiceLink™ Applications

·        Support for the Quick Launch feature for desktop remote sessions in from the Axeda ServiceLink Applications

·        Support for intercepting the beginning and ending of remote sessions through new callbacks

·        Support for reporting the status of managed devices as online or offline in the ServiceLink Applications

·        New API function: AeDRMGetServerStatus()

In addition, the remote session functionality of Agent Embedded was restructured to provide support for the Axeda® Global Access Server's Direct Connect transport mechanism. Support for SSL with Direct Connect has also been added. You can now use GAS Direct Connect on port 443 (the default SSL port) so that your customers do not need to open outgoing or non-standard ports on their firewalls.

Finally, support for Microsoft .NET 2003 is discontinued with the Build 355 of the 6.1 release of Agent Embedded.

Using this Build

To install this build of Agent Embedded:

 

1.     Extract the content of the AgentEmbedded-6.1-364.tar.gz to your current installation directory for Agent Embedded. When prompted whether to overwrite the files, answer Yes.

2.     From your IDE, recompile your project with the new Agent Embedded libraries.
