Introduction

Release 6.6, build 502, of Axeda Agent Embedded provides updates for 64-bit platforms. It also provides updated examples. The following API change should be noted for this update:

Timestamp members in AeTimeValue and AeFileStat types (in Include/AeTypes.h) are now declared as time_t (instead of AeInt32).

This build contains fixes for the following issues:

IDs

Case ID

Description

CONN-1648

97756

Connection through Agent Embedded remote session fails when remote endpoint is local

 

When connecting to the remote session endpoint, Agent Embedded expects the underlying connection operation to be pending before it succeeds or fails. However, on some systems connection operations that are carried out over loopback interfaces can complete immediately. When this happens, the connection fails.

 

RESOLUTION: This build resolves this issue. Specifically, to fix the problem, the AeRemoteChannel state machine is put into the AeRemoteChannelConnecting state before initiating the connection operation.

BTJ-4694

 91987

Agent Embedded sends device status messages that the Axeda Enterprise Server does not handle correctly

 

There are two issues here.
Issue 1: Agent messages for an Agent Embedded agent do not send enough context of the situation for the Axeda Enterprise Server to be able to properly interpret the message. A device status message is used to communicate the status of communication between it and its gateway asset. The Agent Embedded message only mentions the managed asset and not its managing gateway asset.

 

Issue 2: Agent Embedded sends device status messages for the gateway asset itself. Since device status messages are meant to communicate the connectivity status between a managed asset and its managing gateway, it makes no sense for the gateway to be sending device status messages for itself.

 

RESOLUTION: This build resolves these issues. Specifically,

 

Issue 1: To allow efficient message processing on the server, Gateway identification attributes have been added to eMessages that contain one or more Device State messages.

 

Issue 2: To avoid message processing error on the server side, Agent Embedded will not send Device State message on behalf of the "Gateway" device.

 

This build also contains fixes from release 6.5, build 402:

IDs

Case ID

Description

4625 (CONN-942)

93090

OS time changes can cause Agent Embedded to stop communication

 

Because Agent Embedded uses absolute system time for internal time keeping and scheduling of various activities, a non-monotonic change of system time was directly affecting time-based Agent functions. Specifically, setting the system clock back could cause the Agent to delay all activities until the time before the clock change. Instead of absolute system time, Agent Embedded should be using a time source that is not affected by system clock changes.

 

RESOLUTION: To address the problem, Agent Embedded now uses a newly introduced system-dependent function, AeGetElapsedTime(). The function retrieves the amount of time that has elapsed since an arbitrary  point in the past (for example, system start-up time). This point must not change from one invocation of AeGetElapsedTime() within the process to another.

 

An implementation of AeGetElapsedTime() has been added to the default set of

system-dependent functions for both Windows (Sysdeps/Win32/AeOS.c) and Unix

(Sysdeps/Unix/AeOS.c). The function has been added to the set of System-Dependent Functions in the Axeda Agent Embedded Developers Reference.

Supported Platforms

Agent Embedded is distributed with build files for Unix and Microsoft Visual Studio. Built on ANSI C, Agent Embedded can be used on most targets for which an ANSI C compiler is available.

Using this Build

To install this build of Agent Embedded:

 

1.     Extract the content of the AgentEmbedded-6.6-502.tar.gz to your current installation directory for Agent Embedded. When prompted whether to overwrite the files, answer Yes.

2.     From your IDE, recompile your project with the new Agent Embedded libraries.
